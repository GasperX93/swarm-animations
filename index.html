How Swarm works

Visual explanations of the Swarm decentralised storage network, based on the Book of Swarm.

Book of Swarm ↗ ethswarm.org ↗
01 Network Topology

Traditional internet funnels traffic through centralized intermediaries. Swarm routes data peer-to-peer with no single point of control.

02 Push Syncing

Chunks are pushed outward through the network to the neighbourhood responsible for their address.

04 Pull Syncing

Nodes request missing chunks from peers to fill gaps in their local store.

05 Chunk Retrieval

Requests are forwarded via Kademlia routing to the closest node, which returns the chunk along the same path.

06 Postage Stamps

Every chunk requires a postage stamp — proof of payment that funds its storage on the network.

07 Erasure Coding

Parity chunks enable file reconstruction even when some chunks are lost from the network.

08 PSS Messaging

Trojan chunks carry encrypted messages routed to target neighbourhoods — enabling private communication.

09 ACT Access Control

The Access Control Trie manages encryption keys — granting and revoking access to content.

10 Feeds

Single-owner chunks form updateable pointers on immutable storage — each version points to the latest content.
